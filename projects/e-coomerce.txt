5️⃣ Database Design (Schema Details)
🟢 5.1 AUTH SERVICE (MySQL)
Table: users
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    phone VARCHAR(15) UNIQUE,
    email VARCHAR(100) UNIQUE,
    password VARCHAR(255),
    role VARCHAR(20),
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
Table: otp_verification
CREATE TABLE otp_verification (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    phone VARCHAR(15),
    otp VARCHAR(6),
    expiry_time TIMESTAMP
);
🟢 5.2 PRODUCT SERVICE (MySQL)
categories
CREATE TABLE categories (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100),
    description TEXT,
    image_url VARCHAR(255)
);
products
CREATE TABLE products (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(255),
    description TEXT,
    price DECIMAL(10,2),
    category_id BIGINT,
    image_url VARCHAR(255),
    is_active BOOLEAN,
    FOREIGN KEY (category_id) REFERENCES categories(id)
);
🟢 5.3 INVENTORY SERVICE (MySQL)
CREATE TABLE warehouses (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100),
    location VARCHAR(255)
);
CREATE TABLE inventory (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    product_id BIGINT,
    warehouse_id BIGINT,
    stock INT,
    reserved_stock INT DEFAULT 0
);
🟢 5.4 CART SERVICE (Redis)

Key Pattern:

cart:userId

Structure (JSON):

{
  "userId": 101,
  "items": [
    {
      "productId": 12,
      "quantity": 2,
      "price": 50
    }
  ],
  "totalAmount": 100
}
🟢 5.5 ORDER SERVICE (MySQL)
CREATE TABLE orders (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT,
    total_amount DECIMAL(10,2),
    status VARCHAR(50),
    payment_status VARCHAR(50),
    delivery_address TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
CREATE TABLE order_items (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_id BIGINT,
    product_id BIGINT,
    quantity INT,
    price DECIMAL(10,2),
    FOREIGN KEY (order_id) REFERENCES orders(id)
);
🟢 5.6 PAYMENT SERVICE (MySQL)
CREATE TABLE payments (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_id BIGINT,
    transaction_id VARCHAR(255),
    payment_method VARCHAR(50),
    amount DECIMAL(10,2),
    status VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

6️⃣ Kafka Event Flow
	| Event          | Producer      | Consumer           |
	| -------------- | ------------- | ------------------ |
	| OrderCreated   | Order Service | Inventory, Payment |
	| PaymentSuccess | Payment       | Order, Delivery    |
	| OrderConfirmed | Order         | Notification       |
	| OrderShipped   | Delivery      | Notification       |

	Topic Naming:
		order-created-topic
		payment-success-topic
		inventory-update-topic

7️⃣ Order Placement Flow (Step-by-Step)
User adds items → Redis Cart
User clicks checkout

Order Service:
Validates cart
Creates Order (PENDING)
Publish OrderCreated

Inventory:
	Reserve stock
	Publish StockReserved

Payment Service:
	Process payment
	Publish PaymentSuccess

Order Service: Update status → CONFIRMED

Delivery Service: Assign partner

Notification: Send order confirmation
