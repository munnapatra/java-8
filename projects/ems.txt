Employee Management System
🧱 Services Breakdown
| Service                  | Responsibility                    |
| ------------------------ | --------------------------------- |
| **API Gateway**          | Single entry point, routing, auth |
| **Auth Service**         | Login, JWT, RBAC                  |
| **Employee Service**     | Core employee data                |
| **Department Service**   | Departments management            |
| **Attendance Service**   | Attendance & leave tracking       |
| **Payroll Service**      | Salary & payroll                  |
| **Notification Service** | Email/SMS events                  |
| **Audit Service**        | Activity logging                  |
| **Config Server**        | Centralized config                |
| **Discovery Server**     | Service registry                  |

🧩 2. Microservices Design
1️⃣ API Gateway
	Technology: Spring Cloud Gateway
	Responsibilities: JWT validation, Routing, Rate limiting, CORS, Request logging

2️⃣ Auth Service
	Functionalities: 
	User registration, Login, Role-based access control (RBAC), Password reset, Token refresh, 
	Roles: ADMIN, HR, MANAGER, EMPLOYEE

Database Schema
users
------
	id (PK)
	username (unique)
	email
	password_hash
	status (ACTIVE/INACTIVE)
	created_at
	updated_at

roles
------
	id (PK)
	name (ADMIN/HR/MANAGER/EMPLOYEE)

user_roles
-----------
	user_id (FK)
	role_id (FK)

APIs
| Method | Endpoint       | Description   |
| ------ | -------------- | ------------- |
| POST   | /auth/login    | Login         |
| POST   | /auth/register | Register      |
| POST   | /auth/refresh  | Refresh token |
| GET    | /users         | List users    |

5️⃣ Attendance Service

-----------
	id (PK)
	employee_id

	check_in_time
	check_out_time
	status (PRESENT, ABSENT, WFH)

leave_requests
---------------
	id (PK)
	employee_id
	leave_type (SICK, CASUAL, ANNUAL)
	start_date
	end_date
	reason

7️⃣ Notification Service
Event-driven using Kafka.
Events: Employee created, Leave approved, Salary generated, Password reset
Channels: Email, SMS, In-app notifications

🔐 Security Architecture
JWT authentication
Role-based authorization
Method-level security
Password hashing (BCrypt)
Refresh tokens
API rate limiting

🗃 Database Strategy Each service has its own database (Database per service pattern).

| Employee   | Mysql   |
| Payroll    | Mysql   |
| Attendance | Mysql   |

🔄 Communication Strategy
Sync: REST (Feign Client)
Async: Kafka events
Example: Employee created → Publish event → Payroll & Notification consume.
