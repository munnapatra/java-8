What is normalization? Explain 1NF–5NF.
=================================================================================================================================
Normalization: Process of organizing data to reduce redundancy and avoid anomalies (insert/update/delete).

🔹 1NF (First Normal Form)
No repeating groups, all values are atomic
Each column holds single value

👉 Example:
❌ Phone: 123,456
✅ Separate rows for each phone

🔹 2NF (Second Normal Form)
Must be in 1NF
No partial dependency (non-key fully depends on whole primary key)

👉 Used when composite key exists

🔹 3NF (Third Normal Form)
Must be in 2NF
No transitive dependency (non-key depends only on key)

👉 Example:
EmpID → DeptID → DeptName ❌
Fix: Separate Dept table

🔹 BCNF (Boyce-Codd Normal Form)
Stronger than 3NF
Every determinant must be a candidate key

👉 Fixes edge cases not handled in 3NF

🔹 4NF (Fourth Normal Form)
No multi-valued dependencies

👉 Example:
Student → multiple courses & hobbies
Split into separate tables

🔹 5NF (Fifth Normal Form)
No join dependency
Data cannot be further decomposed without loss

👉 Used in complex many-to-many relationships
