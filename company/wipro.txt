How do you explain Java’s multithreading model to someone new to concurrency?
--------------------------------------------------------------------------------------
1. Thread: A thread is an independent path of execution within a program. Java allows multiple threads to run concurrently.

2. Two common ways of Thread creation:
	   Extend `Thread` class
	   Implement `Runnable` (preferred)

3. Thread Lifecycle: `New → Runnable → Running → Waiting/Blocked → Terminated`

4. Concurrency vs Parallelism
	   Concurrency: Multiple tasks progress together (CPU switches between threads).
	   Parallelism: Multiple tasks run at the same time on multiple cores.

5. Thread Coordination / Synchronization
   To avoid race conditions when multiple threads access shared data: `synchronized`, `Lock`, `volatile`, `Atomic` classes

6. Thread Management Utilities
   Java provides high-level APIs in `java.util.concurrent`:
		   `ExecutorService` (thread pools)
		   `Future / CompletableFuture`
		   `BlockingQueue`

7. Best Practice
   Prefer ExecutorService + thread pools instead of manually creating threads for better scalability and resource management.

Beyond the basics, what are impactful use cases of functional interfaces in Java?
--------------------------------------------------------------------------------------
1. Lambda Expressions (Cleaner Code)
   Functional interfaces enable lambdas to replace anonymous classes. Example: `Runnable`, `Comparator`.

2. Stream API Operations
   Used heavily in streams for functional-style data processing.

   `Predicate` → filtering
   `Function` → mapping
   `Consumer` → iteration
   `Supplier` → lazy value generation

3. Strategy Pattern Implementation
   Behavior can be passed as a function instead of creating multiple classes.
   Improves flexibility and reduces boilerplate.

4. Event Handling / Callbacks
   Functional interfaces simplify callback implementations in async or event-driven systems.

5. Asynchronous Programming
   Used with `CompletableFuture` for non-blocking workflows (`thenApply`, `thenAccept`, etc.).

6. Custom Business Rules / Validation
   Define reusable rule logic like `Predicate<T>` for filtering or validating domain objects.

7. Dependency Injection of Behavior
   Allows injecting logic (not just objects), improving testability and configurability.

Key Benefit:
Functional interfaces enable functional programming style, cleaner APIs, and more flexible system design.

Q2. How do you accumulate heap to the jvm and what values do you pass?
--------------------------------------------------------------------------------------
Ans. To accumulate heap to the JVM, specify the initial and maximum heap size using -Xms and -Xmx flags respectively.
Use -Xms flag to specify the initial heap size, e.g. -Xms512m for 512 MB.
Use -Xmx flag to specify the maximum heap size, e.g. -Xmx1024m for 1 GB.

Example: java -Xms512m -Xmx1024m YourClassName
